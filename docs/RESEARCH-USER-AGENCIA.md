## TL;DR — Qué quiere ver la agencia

No quiere una plataforma. Quiere **llegar al viernes con prueba**.

1. **Post-placa:** *"Salió · 229 mil mirando · acá está el video"* — listo para reenviar.
2. **Pre-próxima pauta:** *"En fintech esta semana pautó MP en NDN · vos repetí Olga · no vuelvas a YOL a las 23"* — sin abrir Excel.
3. **Si hay rival:** *"IOL 9 placas vs MP 2 · 44% del rubro"* — una barra, no un informe.

El idioma ya lo tienen. Nosotros no inventamos palabras: **placa, PNT, concurrentes, pico, valle, rubro, Olga, Luzu, zócalo, Tier, Seenka, rate card**.

## 1. Personas

### 1.1 Lucía — Economic buyer + usuario principal

| | |
|---|---|
| **Rol** | Directora de cuentas / socia, agencia 5–30 personas |
| **Clientes** | 2–8 marcas con pauta en Olga/Luzu/Blender |
| **Stack hoy** | WhatsApp con comercial del canal · Seenka · screenshots · Canva/PPT |
| **Ritmo** | Reunión semanal o quincenal de resultados con cada cliente |

**Frase que la define:**

> *"El canal dice que salió perfecto. El cliente me pregunta cuánta gente miraba. Yo mando un link con timestamp más o menos."*

**Qué la hace pagar:** ahorrar 3–5 h del junior armando el deck + ganar credibilidad en la reunión + no depender del comercial de Olga.

**Qué la hace cerrar la pestaña:** otra plataforma con login, 12 tabs, "inteligencia", onboarding de 20 min.

---

### 1.2 Martín — Champion (planner / AE)

| | |
|---|---|
| **Rol** | Planner de medios o AE que arma la reunión del viernes |
| **Motivación** | Verse pro · tener el dato antes que el director |
| **Poder** | Trae el proveedor · no firma el cheque |

**Frase:**

> *"Si me llega el miércoles a la noche con el número y el link, el jueves armo el mail al cliente y el viernes solo presento."*

**Comportamiento observado (proxy demo):** copia solo si el botón dice "Copiar para WhatsApp". Ignora rankings largos. Pregunta "¿y para nuestra marca?" en los primeros 2 minutos.

---

### 1.3 Cliente final (off-screen)

No usa ECO. Recibe el **reenvío de Lucía**.

**Lo que necesita entender en 5 segundos:**

- Su marca salió en [programa] · [canal] · [fecha]
- [N] personas mirando en ese momento
- Link al video (prueba)

**Lo que NO quiere:** jerga de producto, metodología AVE, disclaimers largos.

## 3. Workflow real (no el ideal)

```
Lunes–Miércoles     Jueves              Viernes
──────────────      ───────             ───────
Canal confirma  →   Arman mail/deck  →  Reunión cliente
por WhatsApp        con lo que tienen     "¿rindió?"
     │                    │                    │
     └─ Seenka (¿salió?)  └─ Screenshot        └─ Cliente pide
        a veces              + link aprox           concurrentes
```

**Dónde entra ECO:**

```
Placa sale → [alerta/mensaje] → Lucía copia → WhatsApp al cliente
                    ↓
              Si tiene 5 min más → mira mercado para la próxima
                    ↓
              Si hay rival → mira share antes del viernes
```

**Insight clave:** la web es **backup con link al segundo**. El entregable primario es el **mensaje copiable**.

## 4. Vocabulario — usar SIEMPRE el de ellos

### 4.1 Palabras que ya usan (✅ UI)

| Término | Significado para la agencia | Ejemplo en UI |
|---------|----------------------------|---------------|
| **Placa / PNT** | Mención patrocinada en vivo | "3 placas esta semana" |
| **Concurrentes / mirando** | Gente viendo en ese momento | "229 mil mirando" |
| **Pico / valle** | Mejor/peor momento del programa | "Salió flojo · 12% del pico" |
| **Rubro / categoría** | Vertical (fintech, bebidas…) | "En fintech esta semana…" |
| **Canal / programa** | Olga, Luzu, NDN, Sería Increíble | "IOL · Luzu · NDN" |
| **Tier 1/2/3** | Profundidad de integración | Badge "Tier 2" |
| **Zócalo** | Pie de pantalla en la PNT | Solo en copy largo si aplica |
| **Rate card** | Precio de lista del canal | No mostrar en UI agencia |
| **Orgánico** | Mención no paga | "El rival salió orgánico" (backlog) |
| **Cliente / competidor** | Roles de marca | Badges discretos |
| **Reunión del viernes** | Contexto temporal implícito | "Para reenviar al cliente" |

### 4.2 Palabras que NO usar (❌)

| Evitar | Por qué | Decir en su lugar |
|--------|---------|-------------------|
| Guard, Pulso, Donde | Nombres internos | ¿Rindió la placa? / ¿Dónde pautar? / ¿Quién ganó? |
| Dashboard, plataforma | Suena a más login | "Esta semana en stream" |
| Inteligencia, insights | Buzzword vacío | "Qué hace el mercado" |
| Corpus, export, pipeline | Técnico | "Esta semana" + fecha |
| Attention, CPM (en hero) | Metodología, no operación | "mirando" + número |
| Fit score, planner | Custoo territory | "Conviene entrar acá" |
| Vitamin, wedge | Interno fundador | — |

### 4.3 Frases que resuenan (validadas en research)

- *"¿Cuánta gente miraba?"*
- *"¿Rindió la pauta?"*
- *"Seenka confirma que salió — nosotros cuántos miraban."*
- *"No repitas ese horario."*
- *"¿Qué hizo Mercado Pago esta semana?"*
- *"Pasame algo que le mande al cliente."*

## 5. Mental models — cómo procesan la data

### 5.1 Jerarquía de confianza

```
1. Link YouTube al segundo exacto     ← prueba irrefutable
2. Número grande de concurrentes      ← el dato que Seenka no da
3. Cita textual del conductor         ← "esto dijeron en vivo"
4. Seenka (aparición sí/no)           ← complemento, no reemplazo
5. Mail del comercial del canal       ← sesgo, no independiente
```

**Implicación UI:** número hero + cita + botón YouTube **siempre juntos**. Nunca un número solo sin link.

### 5.2 Qué miran primero (eye-tracking proxy — orden de demo)

1. **El número grande** (229k mirando)
2. **Nombre marca + canal + programa**
3. **Botón copiar** (¿lo puedo mandar ya?)
4. **Badge "salió flojo"** (¿hubo problema?)
5. Todo lo demás (rankings, matrices) — secundario

### 5.3 Comparaciones que entienden

| Entienden | No entienden / no les importa |
|-----------|-------------------------------|
| Cliente vs rival (2 columnas) | Ranking 135 marcas |
| % del pico del programa | avg_logprob, tiers metodológicos |
| Share del rubro (barra simple) | Valor USD sin contexto |
| "9 placas vs 2 placas" | Gráficos de línea multi-serie |
| 8 canales con escala/nicho | Matriz heatmap |

### 5.4 Tolerancia a limitaciones

Prefieren **honestidad visible** a promesas rotas:

- *"Luzu no tiene chat esta semana"* → OK
- *"Una semana de data"* → OK
- *"Green Life no es rival OTC de Geniol"* → OK (ejemplo armado)
- Datos inventados o vacíos sin explicación → cierran

## 7. Competencia en su cabeza

| Herramienta | Qué resuelve | Qué NO resuelve (gap ECO) |
|-------------|--------------|---------------------------|
| **Seenka** | ¿Salió la pauta? ¿Cuántas veces? | Concurrentes al segundo, cita, link VOD |
| **WhatsApp Olga/Luzu** | Reservar slot, confirmar salida | Dato independiente, comparable |
| **Custoo / StreamsRating** | Audiencia de programas | Minuto de la placa, chat, competencia |
| **Screenshot YouTube** | Prueba visual aproximada | Timestamp exacto, escala numérica |
| **Junior + Canva** | Deck presentable | 5 horas, no escala |

**Posicionamiento en UI (una línea, no banner):**

> *"Seenka confirma que salió. Acá ves cuánta gente miraba en ese segundo."*

## 10. Implicaciones para UX/UI (puente al spec)

1. **Pregunta = título** — no features.
2. **Un número hero por bloque** — el resto es contexto.
3. **Copiar > exportar > explorar** — orden de botones.
4. **Cliente primero, rival colapsado** — siempre.
5. **Una marca en foco** — barra "Mirando ahora".
6. **Story rail** — placa → mercado → rival (opcional).
7. **Límites visibles** — semana, sin chat, post-programa.
8. **Cero onboarding** — elegir marca y listo.
